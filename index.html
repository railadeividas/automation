Šis puslapis buvo sukurtas ir deploy'intas į GitHub Pages naudojant Hermes Agent (AI asistentą). Toliau — pilnas visų žingsnių aprašymas su komandomis, kurios buvo vykdomos.

📋 Ko reikia prieš pradedant

  • GitHub token'as — susikurk čia. Rekomenduojama: classic PAT su repo scope. ⚠️ NIEKADA nesiųsk token'o į repozitoriją!

1. Repozitorijos paruošimas

1 Apsirašome GitHub token'ą
# Token'ą saugome tik lokaliai, NIEKADA ne push'inam į repozitoriją!
# Įrašome į git credential helper:
git config --global credential.helper store
echo "https://${GITHUB_USER}:${GITHUB_TOKEN}@github.com" >> ~/.git-credentials
⚠️ SVARBU: Token'as lieka tik tavo kompiuteryje (~/.git-credentials). Jis NĖRA ir NIEKADA NEBUS push'intas į GitHub repozitoriją!
2 Patikriname ar token'as veikia
# Užklausa į GitHub API:
curl -s -H "Authorization: token ${GITHUB_TOKEN}" \
  https://api.github.com/user | jq .login
# Atsakymas: "railadeividas" ✅

# Patikriname teises į repozitoriją:
curl -s -H "Authorization: token ${GITHUB_TOKEN}" \
  https://api.github.com/repos/railadeividas/automation | \
  jq '{permissions: .permissions}'
# {"permissions": {"admin": true, "push": true, ...}} ✅

2. Repozitorijos padarymas viešos (public)

3 Pakeičiam vizualumą į public

GitHub Pages nemokamai veikia tik ant public repozitorijų (arba su GitHub Pro).

curl -s -X PATCH \
  -H "Authorization: token ${GITHUB_TOKEN}" \
  -H "Accept: application/vnd.github+json" \
  https://api.github.com/repos/railadeividas/automation \
  -d '{"private": false}'
# Repo pakeista į public ✅
4 Klonuojame repozitoriją
git clone https://github.com/railadeividas/automation.git
cd automation

4. Failų įkėlimas į GitHub per API

ℹ️ PASTABA: Šiame žingsnyje git push neveikė, nes buvo naudojamas fine-grained PAT. Tokie token'ai neveikia su git HTTPS push operacijomis, nors API užklausos veikia puikiai. Sprendimas — įkelti failus per GitHub REST API.
7 Įkeliame failus per API (PUT /contents)
# Python script'as failų įkėlimui per API:
import json, urllib.request, base64

def create_file(path, content, message):
    data = json.dumps({
        "message": message,
        "content": base64.b64encode(content.encode()).decode(),
        "branch": "master"
    }).encode()
    req = urllib.request.Request(
        f'https://api.github.com/repos/railadeividas/automation/contents/{path}',
        data=data,
        headers={'Authorization': f'token {TOKEN}'},
        method='PUT'
    )
    resp = json.loads(urllib.request.urlopen(req).read())
    return resp['content']['sha']

# Įkeliame index.html
sha1 = create_file("index.html", html_content,
                   "Initial commit: index.html")
# Atsakymas: SHA 4713c3e0 ✅

# Įkeliame README.md
sha2 = create_file("README.md", md_content,
                   "Initial commit: README.md")
# Atsakymas: SHA fd66bbe0 ✅

5. GitHub Pages įjungimas

8 Įjungiame Pages per API
# POST request sukuria Pages konfigūraciją:
curl -s -X POST \
  -H "Authorization: token ${GITHUB_TOKEN}" \
  -H "Accept: application/vnd.github+json" \
  https://api.github.com/repos/railadeividas/automation/pages \
  -d '{"source": {"branch": "master", "path": "/"}}'
# Atsakymas: {"status": "building", "html_url": "https://railadeividas.github.io/automation/"} ✅
9 Tikriname būseną
# Laukiame kol GitHub Pages su(si)deploy'ins (apie 1-2 min):
curl -s -H "Authorization: token ${GITHUB_TOKEN}" \
  https://api.github.com/repos/railadeividas/automation/pages | \
  jq '.status'
# Pirma: "building" → po kelių bandymų: "built" ✅

# Patikriname ar puslapis pasiekiamas:
curl -s -o /dev/null -w "%{http_code}" \
  https://railadeividas.github.io/automation/
# Atsakymas: 200 ✅ — puslapis gyvas!

🚨 Dažniausios klaidos ir sprendimai

🔴 git push neveikia su fine-grained PAT

Klaida: remote: Invalid username or token. Password authentication is not supported for Git operations.

Sprendimas:

  • 1 variantas: Naudoti classic PAT su repo scope — jis veikia ir su git push, ir su API
  • 2 variantas: Naudoti GitHub REST API PUT /repos/{owner}/{repo}/contents/{path} — veikia su bet kokiu token'u
  • 3 variantas: Nusistatyti SSH raktą ir naudoti git@github.com:...

🔴 GitHub Pages neatsidaro (404)

Sprendimas: Palaukti 1-2 minutes, kol GitHub sudeploy'ins. Taip pat įsitikinti kad repozitorija yra public ir kad yra index.html failas šakninėje.

🔴 Token'as netyčia pateko į repozitoriją

⚠️ JEI TAIP ATSITIKO: Nedelsiant ištrink token'ą čia! Tada sukurk naują.

📚 Komandų cheat sheet

# === Git ===
git config --global user.name "Vardas"
git config --global user.email "email@example.com"
git clone https://github.com/owner/repo.git
git add . && git commit -m "žinutė"
git push origin master

# === GitHub API ===
# Patikrinti token'o teises:
curl -s -H "Authorization: token ${TOKEN}" https://api.github.com/user

# Padaryti repo public:
curl -X PATCH -H "Authorization: token ${TOKEN}" \
  https://api.github.com/repos/owner/repo -d '{"private": false}'

# Sukurti failą per API:
curl -X PUT -H "Authorization: token ${TOKEN}" \
  https://api.github.com/repos/owner/repo/contents/index.html \
  -d '{"message":"commit","content":"'$(base64 <<< "HTML")'","branch":"master"}'

# Įjungti GitHub Pages:
curl -X POST -H "Authorization: token ${TOKEN}" \
  https://api.github.com/repos/owner/repo/pages \
  -d '{"source":{"branch":"master","path":"/"}}'

# Patikrinti ar puslapis gyvas:
curl -s -o /dev/null -w "%{http_code}" https://user.github.io/repo/

🔑 Token'o saugumas (svarbiausia!)

⛔ NIEKADA:
  • Nedėk token'o į failus, kurie push'inami į GitHub (.env, konfigūracijos failai, kodo failai)
  • Nerašyk token'o į index.html, README.md ar bet kokį kitą viešą failą
  • Nesiųsk token'o per nesaugius kanalus (email, SMS, public chat)
✅ GERAI:
  • Saugok token'ą lokaliai: ~/.git-credentials, ~/.bashrc eksportas, ~/.hermes/.env
  • Naudok GitHub Actions secrets jei token'o reikia CI/CD pipeline'uose
  • Jei token'as nutekėjo — ištrink jį ir sukurk naują
